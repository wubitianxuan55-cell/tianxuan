# tianxuan V6.1.0 Release Notes

> 发布日期: 2026-06-12 | 基于 V5.25 | Go 1.25 | DeepSeek V4 | 参考 MiMo-Code

---

## 概述

V6.1 是一次重大升级，核心目标是**降低模型认知负担**和**提升上下文管理效率**。参考 MiMo-Code、Claude Code、Reasonix、Codex 等编码 Agent 的设计精华，新增预算制上下文注入、3 Agent 模式、Judge 模型验证、Dream/Distill 自我进化、会话归档等能力。

## 新增功能

### P1–P4 运行时增强

| 功能 | 说明 |
|------|------|
| **并行工具默认化** | 工具按冲突键分组批量执行，编辑不同文件可并行 |
| **子代理轻量化** | TemplatePrefix + ActiveSchemas 共享前缀缓存，成本降 19% |
| **3 Agent 模式** | `/mode explore/develop/orchestrate` 三种运行模式 |
| **缓存友好压缩** | pressure_flush + output_continue + repeat_detect + stop_gate |

### P5–P8 智能进化

| 功能 | 说明 |
|------|------|
| **Budgeted 上下文注入** | 4 组件重要性权重预算：checkpoint(0.9) 30% / memory(0.7) 15% / tasks(0.5) 10% / recent(1.0) 45% |
| **Dream & Distill** | `/dream [scan\|extract]` 知识提取，`/distill [scan\|create]` 模式发现 + skill 生成 |
| **Goal/Stop 验证** | `/goal` 设置目标，stopGate 模式感知强制验证 |
| **工具合并 41→25** | `[tools] compact = true` 隐藏冗余工具，降低模型认知负担 |

### 死代码清理

移除 ~800 行遗留代码：GoalRouter、FirstTurnHandler（已被 V3.0 ContextManager 完全替代）

## 新增命令

| 命令 | 功能 |
|------|------|
| `/mode e/dev/o` | 切换 explore/develop/orchestrate 模式 |
| `/goal <desc>` | 设置会话停止条件 |
| `/dream [scan\|extract]` | 跨会话知识提取与分析 |
| `/distill [scan\|create]` | 模式发现与 skill 自动生成 |
| `/memories [query]` | 搜索持久记忆 |

## 配置新增

```toml
# tianxuan.toml
[tools]
compact = true  # V6.0 P8: 启用精简工具集（~25 个可见工具）
```

### V6.1 新增 (MiMo-Code 启发)

| 功能 | 说明 |
|------|------|
| **Judge 模型** | 独立 flash API 调用验证 `/goal` 条件，消除乐观停止 |
| **Checkpoint-Writer** | 预算重建后自动 QueueMemory 摘要 |
| **树形任务** | T1/T1.1 格式提示注入 nudge（不修改 tool schema） |
| **会话归档** | JSONL archive 跨会话查询 + 工具使用统计 |

## 升级指南

从 V5.25 升级：

```bash
cd tianxuan
git checkout main
git pull
go build -o bin/tianxuan.exe ./cmd/tianxuan
```

无需配置变更即可运行，V6.0 所有新功能默认关闭（向后兼容）。按需启用：

```bash
# 启用精简工具集
echo "[tools]\ncompact = true" >> tianxuan.toml

# 尝试 3 Agent 模式
/mode explore
```
