## 三、关键类型系统

### 3.1 AgentRunner — 核心执行器

```go
// internal/agent/agent.go

type AgentRunner struct {
    prov        provider.Provider          // DeepSeek API 客户端
    tools       *tool.Registry             // 工具注册表 (17内置 + MCP + 技能)
    session     *Session                   // 对话历史
    dispatcher  *ToolDispatcher            // 前置检查 (计划模式/权限/钩子)
    ctxMgr      *tiancontext.ContextManager // TCCA 内核 (nil = 旧模式)

    // 事件和UI
    sink        event.Sink                 // 事件输出流
    gate        Gate                       // 权限门
    hooks       ToolHooks                  // Pre/PostToolUse 钩子
    asker       Asker                      // 交互式问答

    // 压缩 (V5.0: 纯截断)
    compaction  CompactionConfig

    // 后台任务
    jobs        *jobs.Manager
    evidence    *evidence.Ledger
    memQueue    memory.Queue

    // 性能优化 (V4.2)
    preOutcomes map[string]toolOutcome     // 流式预执行结果
    preMu       sync.Mutex
    preWG       sync.WaitGroup
    tc          *toolCache                 // 文件读取缓存

    // 缓存统计
    sessCacheHit  atomic.Int64
    sessCacheMiss atomic.Int64
}
```

### 3.2 Tool — 工具接口

```go
// internal/tool/tool.go

type Tool interface {
    Name() string
    Description() string
    Schema() json.RawMessage           // JSON Schema 参数定义
    Execute(ctx, args) (string, error)  // 执行
    ReadOnly() bool                     // 只读? → 可并行

// V4.2 新增: 紧凑描述 (节省 ~75% prompt token)
type CompactDescriptor interface {
    CompactDescription() string          // 单行中文, ≤25 字
    CompactSchema() json.RawMessage      // 精简 Schema (无描述/约束)
}
```

## 四、数据流详解

### 4.1 单次 Turn 完整流程

```
User Input "增加JWT登录"
  │
  ├─ Controller.Compose()
  │   ├─ ctxMgr.AssemblePrompt()
  │   │   ├─ L1: IdentityLayer.SystemPrompt()     ← 300 tok (prefix cache 100%)
  │   │   ├─ L2: RuntimeLayer.SystemPrompt()       ← 100 tok
  │   │   ├─ L4: FlowLayer.Messages()              ← 对话历史
  │   │   └─ return []provider.Message
  │   └─ → provider.Request{Tools: reg.Schemas()}
  │
  ├─ AgentRunner.Run(ctx, input)
  │   └─ runDirect(ctx, input)
  │       ├─ evidence.Reset() + tc.clear()
  │       ├─ session.Add(UserMessage)
  │       │
  │       ├─ stream(ctx, step) ────────────────────── DeepSeek API ─┐
  │       │   ├─ tools = reg.Schemas()  ← CompactDescriptor 自动选择
  │       │   ├─ msgs = session.Messages + L2 注入
  │       │   ├─ prov.Stream(ctx, Request{msgs, tools})
  │       │   └─ for chunk := range ch:
  │       │       ├─ ChunkReasoning → sink.Emit
  │       │       ├─ ChunkText → sink.Emit
  │       │       ├─ ChunkToolCall → calls 收集
  │       │       │   └─ 只读? → goroutine 预执行 (preWG)    ←── 并行!
  │       │       └─ ChunkUsage → 缓存统计
  │       │
  │       ├─ 无 tool_calls? → return nil (最终回答)            ←── 结束
  │       │
  │       ├─ preWG.Wait()  ← 等待预执行完成
  │       ├─ executeBatch(ctx, calls)
  │       │   ├─ 逐个 emit ToolDispatch
  │       │   ├─ partitionToolCalls → [并行批次 | 串行批次]
  │       │   ├─ run(i):
  │       │   │   ├─ preOutcomes 命中? → 直接返回             ←── 缓存!
  │       │   │   ├─ read_file? → tc.get(path, offset)        ←── 文件缓存!
  │       │   │   ├─ executeOne(ctx, call)
  │       │   │   │   ├─ ToolDispatcher.Check() → 计划模式? 权限?
  │       │   │   │   ├─ onPreEdit (写入工具快照)
  │       │   │   │   └─ t.Execute(ctx, args) → result
  │       │   │   └─ read_file? → tc.set(path, offset, result)
  │       │   │      写入工具? → tc.invalidatePath(path)
  │       │   └─ emit ToolResult + 风暴检测
  │       │
  │       ├─ 结果注入 session
  │       └─ maybeCompact(ctx, usage)  ← 超阈值? 截断到 50 条
  │
  └─ Controller → sink → UI 渲染
```

### 4.2 Prompt 组装 (发送给 DeepSeek 的最终格式)

```
┌─────────────────────────────────────────────┐
│ System: 你是 tianxuan (身份+规则)            │ ← L1 (300 tok)
│ [工具列表: read(path)→内容 | edit(p,s,r)→确认│     prefix cache
│  glob(p)→文件 | grep(p)→行 | ...]           │     100% 命中
├─────────────────────────────────────────────┤
│ [项目: Go · 入口: cmd/main.go · 最近编辑:   │ ← L2 (100 tok)
│  auth.go, user.go]                          │
├─────────────────────────────────────────────┤ ← Prefix Cache 边界
│ [user] 创建用户系统                         │
│ [assistant] 我来创建 user.go...             │ ← L4 对话历史
│ [tool] read_file("main.go") → ...          │    每轮追加
│ ...                                         │
├─────────────────────────────────────────────┤
│ [user] 增加 JWT 登录                        │ ← 当前输入
└─────────────────────────────────────────────┘

首轮: ~450 tok (L1+L2+user) → 实际计费 ~100 tok
N轮后: ~450 + ~2K×N tok → 新增部分计费, prefix 免费
```

## 五、工具系统

### 5.1 内置工具 (17个)

| 工具 | 紧凑描述 | 类型 | 缓存 |
|------|---------|------|------|
| `read_file` | 读取文件内容(可选行范围/分页) | 只读 | ✅ 5s + mtime |
| `edit_file` | 精确替换文件字符串(须全局唯一) | 写入 | 失效 path |
| `write_file` | 写入/覆盖文件(自动建父目录) | 写入 | 失效 path |
| `multi_edit` | 原子化批量编辑(单文件N步依次执行) | 写入 | 失效 path |
| `delete_range` | 删除文件连续行(起止锚点定位) | 写入 | 失效 path |
| `delete_symbol` | 删除Go符号(函数/类型/接口等,AST解析) | 写入 | 失效 path |
| `glob` | 通配符匹配文件名(支持**递归) | 只读 | ❌ |
| `grep` | 正则搜索文件内容(返回path:行:文本,限200条) | 只读 | ❌ |
| `ls` | 列目录条目(子目录带/) | 只读 | ❌ |
| `bash` | 执行shell命令(合并stdout+stderr,限2分钟) | 写入 | ❌ |
| `bash_output` | 读取后台任务的增量输出(不阻塞) | 只读 | ❌ |
| `kill_shell` | 终止后台任务(SIGTERM→SIGKILL) | 写入 | ❌ |
| `wait` | 阻塞等待后台任务结束(可设超时) | 只读 | ❌ |
| `web_fetch` | 抓取URL纯文本(去标签,SSRF安全) | 只读 | ❌ |
| `todo_write` | 更新任务清单(全量替换,最多一个进行中) | 只读* | ❌ |
| `complete_step` | 完成计划步骤(附验证证据,空证据拒绝) | 只读* | ❌ |
| `notebook_edit` | 编辑Jupyter Notebook单元格(.ipynb) | 写入 | ❌ |

> *`todo_write` 和 `complete_step` 虽标记为只读但**不参与**流式预执行和并行批次, 因为它们依赖前序工具的证据账本。

### 5.2 紧凑描述 (V4.2 新增)

所有 17 个内置工具通过 `CompactDescriptor` 接口提供:
- `CompactDescription()`: 单行中文, 15-25 字
- `CompactSchema()`: 精简 JSON (仅 type + properties 名+类型 + required, 无参数描述/约束)

`Registry.Schemas()` 自动检测 `CompactDescriptor` 并使用紧凑版本。

### 5.3 流式预执行 (V4.2)

只读工具的 `executeOne()` 在 `stream()` 收到完整 tool_call 后立即 goroutine 启动。`executeBatch()` 通过 `preOutcomes` map 检查已完成的预执行结果。

排除: `complete_step`, `todo_write` (依赖前序), 空 ID 调用。

### 5.4 文件缓存 (V4.2)

`toolCache` (5s TTL + mtime 校验):
- 键: `path` 或 `path@offset`
- 读命中: 跳过 `Execute()`, 直接返回缓存结果
- 写失效: `edit_file/write_file/multi_edit/delete_range/delete_symbol` 自动清除该路径的所有缓存

## 六、四域缓存架构 (TCCA)

```
┌──────────────────────────────────────────────────┐
│ L1: IdentityLayer (identity.go)                  │
│ "你是 tianxuan..." + 规则 + 配置                  │
│ 字节稳定 → SHA-256 哈希 → 跨会话验证              │
│ ~300 tok · 100% cache 命中                       │
├──────────────────────────────────────────────────┤
│ L2: RuntimeLayer (runtime.go)                    │
│ ProjectState + SessionState + ExecutionState     │
│ 项目语言/模块/入口 + 最近编辑 + 当前目标           │
│ ~100 tok · 会话内稳定 · 跨会话可能变化             │
├──────────────────────────────────────────────────┤
│ L3: SkillLayer (skill.go)                        │
│ 工具定义 + 技能索引 + 子代理描述                   │
│ V5.0: 静态 v1 配置文件, 无 Learner 自适应          │
│ ~1,200 tok (紧凑) · 与 L1 一起 prefix cache       │
├──────────────────────────────────────────────────┤
│ L4: FlowLayer (context/flow.go)                  │
│ 对话历史 · 每轮追加                               │
│ V5.0: 超 500K tok → 纯截断 (保留系统+最后 50 条)   │
│ 可变大小 · 前缀可缓存 · 新增部分不可缓存            │
└──────────────────────────────────────────────────┘
```

## 八、关键决策记录

| 决策 | 原因 | 影响 |
|------|------|------|
| 删除 Planner | DeepSeek V4 推理能力覆盖规划需求; 双模型 +50% API 调用 | -133 行, 节省 50% API 成本 |
| 删除 Learner | 零证据有效; DeepSeek V4 行为一致无需纠偏 | -535 行 |
| 截断替代 Compact | 1M 窗口够用; LLM 摘要成本 = 省下的 token 成本 | 消除 5-30s 阻塞, -457 行 |
| 紧凑工具描述 | JSON Schema token 占 prompt 30-70% | -75% L3 token |
| 流式预执行 | 只读工具在 stream 期间并行 | 多工具延迟 -50% |
| 文件缓存 | 同轮重复读文件浪费 IO | 合并重复读 |

## 九、性能模型

```
单轮成本 = CacheMiss × ¥2/1M + Output × ¥8/1M
         ≈ 100 tok × ¥0.002 + 200 tok × ¥0.008
         ≈ ¥0.0002 + ¥0.0016
         ≈ ¥0.0018/轮

10轮实测: ¥0.0082 (平均 ¥0.0008/轮)
1000轮预算: ¥0.82

瓶颈分析:
  TTFT: 1-5s (DeepSeek API, 不可控)
  Compact: 0s (纯截断, < 1ms)
  Planner: 0s (已删除)
  工具执行: < 100ms (缓存 + 并行)
  感知最快: ~1s
  感知最慢: ~5s (API 延迟)
```
