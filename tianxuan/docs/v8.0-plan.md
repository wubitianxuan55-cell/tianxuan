# tianxuan V8.0 大改方案

基于 DeepSeek-Reasonix V1.9.1 分析，11 项新特性全量实施。

## P0 — DeepSeek 缓存架构放大（~200 行 Go）

### 1. 工具按需过滤（Tool Lazy Loading）
**现状**：stream() 中 `tools := a.tools.Schemas()` 发送全量工具 schema
**改进**：根据当前任务上下文动态精简工具列表
- 识别 task_kind → 只发相关工具
- 例如 review 任务不发 write_file/edit_file/bash 等写入工具
- 影响：减少 prompt 中工具描述体积 ~30%，提高缓存命中率
**文件**：`internal/agent/agent.go` stream()、`internal/cache/skill.go`

### 2. 确定性工具结果剪枝
**现状**：toolCache 仅缓存 read_file 结果（文件 mtime 校验）
**改进**：扩展到所有只读工具，相同参数+相同结果 → 不发重复 token
- glob/ls/grep/git_status/git_diff/git_log 等也加入缓存
- 相同参数返回相同结果的确定性工具，跳过重复发送
**文件**：`internal/agent/toolcache.go` 扩展 key 策略

### 3. Mid-turn Steer 注入
**现状**：模型输出中途无法干预
**改进**：在 streaming 过程中检测到错误方向时注入"系统提示"
- 检测模式：连续空输出 >3 轮、重复相同错误 >2 轮、连续写入被拒绝
- 注入方式：在下一轮用户消息前追加 `[System note: ...]`
- Reasonix 做法：`runDirect` 循环中 after stream 检查，下一轮前注入
**文件**：`internal/agent/agent.go` runDirect() 循环

## P1 — 增强独立性和体验（~200 行 Go + 50 行 TS）

### 4. Plan 模式智能澄清
**现状**：plan mode 只拦截写入工具，模型可自由输出 plan
**改进**：当用户输入过于模糊（<20字符、无明确目标词）时主动追问
- 追问模板：明确目标/期望结果/约束条件
- 用户回答后自动拼接为完整 prompt 再进入 plan mode
**文件**：`internal/control/controller.go` Submit()、`internal/control/input.go`

### 5. read_skill 内置工具
**现状**：Agent 无法主动读取 skill body，只能通过系统 prompt 的 skill index 知道存在
**改进**：新增 `read_skill` 只读工具，Agent 可用它动态读取技能内容
- 参数：`name`（技能名）
- 返回：技能的 frontmatter + body，或 "not found"
**文件**：`internal/tool/builtin/readskill.go`（新文件，~40行）

### 6. PDF 附件文本提取
**现状**：不支持 PDF 文件
**改进**：读取 PDF 文件并提取纯文本
- 使用 Go 标准库或轻量第三方库
- 通过 read_file 扩展或独立工具 `pdf_extract`
**文件**：`internal/tool/builtin/pdfs.go`（新文件，~60行）

### 7. SSH 断开生存（跳过）
**理由**：tianxuan 是桌面应用，本地运行，不存在 SSH 断开场景

## P2 — 锦上添花（~150 行 Go + ~300 行 TS）

### 8. 内置 Context7 MCP 服务器
**现状**：仅有 CodeGraph 内置 MCP
**改进**：新增 Context7 MCP（提供最新库文档上下文）
- 配置在 `[[plugins]]` 或 `.mcp.json`，默认启用
**文件**：`internal/boot/boot.go` 新增 MCP spec

### 9. 命令面板 (Cmd+K)
**现状**：`/` 斜杠菜单仅在输入框聚焦时可用
**改进**：全局 Cmd+K 命令面板（类似 VS Code）
- 列出所有可用命令：技能/模式切换/设置/MCP/记忆/历史等
- 模糊搜索 + 键盘导航
- 复用现有 SlashMenu 组件逻辑
**文件**：`desktop/frontend/src/components/CommandPalette.tsx`（新文件）

### 10. /goal 循环命令
**现状**：无目标分解机制
**改进**：新增 `/goal` 命令将大目标分解为子任务
- `/goal "实现用户登录"` → Agent 分解为子任务
- 每个子任务独立 turn，完成后自动进入下一个
- 进度追踪
**文件**：`internal/control/controller.go` 新命令处理

### 11. 桌面原生通知
**现状**：仅内联 notice 消息
**改进**：Wails 桌面端原生通知
- 长任务完成、审批请求、错误时触发
- 使用 Wails runtime 的 notification API
**文件**：`desktop/frontend/src/hooks/useNotify.ts`（新文件）、`desktop/app.go`

### 12. PermissionRequest Hook
**现状**：仅有 PreToolUse hook（执行前检查）
**改进**：新增 PermissionRequest hook 类型
- 比 PreToolUse 更灵活：可修改参数、可设置 session 级别授权
- 支持自定义审批策略
**文件**：`internal/hook/hook.go` + `internal/hook/permission.go`

### 13. Skill 嵌套发现（跳过）
**理由**：tianxuan 已有 convention dirs 发现（`.claude/skills/`、`.agents/skills/` 等），已达 Reasonix 同等水平
