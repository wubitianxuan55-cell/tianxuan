# V1.6 会话自动恢复 — 设计文档

> **状态**: 待审批
> **分支**: `v1.5-dev`（基于 `v1.5.0` tag）

---

## 1. 目标

桌面端启动时自动恢复上次未完成的会话，用户无需手动操作即可继续工作。

## 2. 行为定义

| 场景 | 行为 |
|------|------|
| 工作区有历史会话 | 自动加载最近一条（按 mtime） |
| 最近会话为空 | 新建空白会话（当前行为） |
| 工作区无会话 | 新建空白会话（当前行为） |
| 会话文件损坏 | fallback 新建会话，日志记录 |
| 工作区切换（不同 cwd） | 每个工作区独立，各自恢复各自的最新会话 |

## 3. 改动清单

### 3.1 `desktop/app.go` — 启动时检测并恢复

仅修改 `buildController()`：在创建 controller 并设置 sessionPath 后，检查工作区是否有历史会话。

```go
func (a *App) buildController() {
    // ... 现有代码：ensureWorkspace, boot.Build, EnableInteractiveApproval ...

    dir := config.WorkspaceSessionDir("")
    ctrl.SetSessionPath(agent.NewSessionPath(dir, ctrl.Label()))

    // V1.6: 尝试自动恢复最近会话
    sessions := agent.ListSessions(dir)
    if len(sessions) > 0 {
        latest := sessions[0] // ListSessions 按 mtime 降序
        if latest.Turns > 0 { // HasContent 的代理判断
            if loaded, err := agent.LoadSession(latest.Path); err == nil {
                ctrl.Resume(loaded, latest.Path)
                slog.Info("auto-resumed session", "path", latest.Path, "turns", latest.Turns)
            }
        }
    }

    // ... 其余不变 ...
}
```

**关键点**：
- `ListSessions` 已存在（`agent/save.go`），按 mtime 降序返回
- `LoadSession` 已存在（`agent/save.go`），返回错误时安全 fallback
- `ctrl.Resume` 已存在，会设置 startedOnce → SessionStart hook 正确触发
- SessionPath 已先设为新路径，Resume 会覆盖

### 3.2 无其他文件改动

`agent/save.go`、`control/controller.go`、`config/config.go` 均不需要修改——所需 API 全部已存在。

## 4. 不影响缓存架构

- L1 系统提示词：Resume 加载的是对话历史（messages[1:])，L1 由 `agent.NewSession(sysPrompt)` 在启动时新建——与 V1.5 完全相同
- 会话文件：JSONL 只存 conversation messages，不存系统提示词
- 检查点：`ctrl.Resume` 调用 `rebindCheckpoints`，正确加载 .ckpt/ 目录

## 5. 验收标准

- [ ] 首次启动：工作区无会话 → 空白新会话
- [ ] 二次启动：自动加载上次会话，对话历史完整可见
- [ ] 空会话不恢复：上次只发送了"你好"然后关闭 → 第二次启动仍是空白会话
- [ ] 切换工作区：打开不同项目 → 各自独立恢复
- [ ] 会话文件损坏：启动不崩溃，fallback 新会话
- [ ] `go build` 零错误
- [ ] 现有测试全部通过
