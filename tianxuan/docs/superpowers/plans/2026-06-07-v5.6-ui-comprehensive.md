# V5.6 UI 全面优化计划

> **给 agentic worker：** 使用本计划执行。步骤使用 checkbox（`- [ ]`）语法跟踪。

**目标：** 三合一 UI 优化 — CSS 模块化拆分 + 交互体验修复 + 视觉打磨统一。

**架构：** 分 3 个阶段独立执行，每阶段产出可验证。保持现有 BEM 类名不变（零 JSX 修改），仅拆分 CSS 文件 + 修复样式缺陷。CSS 变量体系合并（tokens.css + styles.css → 统一 tokens.css）。

**技术栈：** Vite 6 + React 18 + 纯 CSS（保持现有方案，不引入 Tailwind/CSS-in-JS）。

## 阶段 1：CSS 架构重构

### 现状
- `styles.css` 5196 行单文件，所有组件样式混在一起
- `tokens.css` 有独立的 `--color-*` 变量体系，与 `styles.css` 的 `--bg`/`--fg` 体系**不兼容**，存在两套变量
- `Transcript.module.css` 21 行死代码（无人导入）
- `main.tsx` 仅导入 `styles.css`

### 拆分策略（保持 BEM 类名不变）

| 新文件 | 内容 | 来源行号估算 |
|--------|------|-------------|
| `base.css` | CSS reset、`:root` 变量、基础排版、滚动条、通用工具类（`.ico`、`.spin`、`.phase`、`.notice`） | ~1-100, ~4900-5196 |
| `app.css` | App 布局（`.app`、`.sidebar`、`.main`）、StatusBar、UpdateBanner | ~100-400 |
| `transcript.css` | `.transcript`、`.msg`、`.reasoning`、`.compaction`、`Welcome` | ~400-1000 |
| `tool.css` | `.tool`、ToolGroup、Diff/Code 查看器、CopyButton | ~1000-1800 |
| `composer.css` | `.composer`、`.slash`、`.filemenu`、`.argmenu` | ~1800-2400 |
| `panels.css` | SettingsPanel、HistoryPanel、MemoryPanel、CapabilitiesPanel 等侧栏面板 | ~2400-4900 |

### 任务列表

#### 1.1 删除死代码
- 删除 `Transcript.module.css`（21 行，无人导入）

#### 1.2 统一 CSS 变量 → tokens.css
- 将 `styles.css` 的 `:root` 变量（`--bg`/`--fg`/`--accent`/`--border` 等 ~50 个变量）合并到 `tokens.css`
- 删除 `tokens.css` 中未被使用的 `--color-*` 变量（`search_content` 确认哪些被使用）

## 阶段 2：交互体验修复

### 2.1 长文本溢出修复
**文件：** `transcript.css`（拆分后）

给 `.msg__bubble` 添加：
```css
.msg__bubble {
  word-break: break-word;
  overflow-wrap: break-word;
  min-width: 0; /* 防止 flex 子元素溢出 */
}
```

同样给 `.reasoning__body` 添加：
```css
.reasoning__body {
  word-break: break-word;
  overflow-wrap: break-word;
}
```

### 2.2 折叠动画平滑化
**文件：** `tool.css`、`transcript.css`

当前 ToolCard 折叠使用条件渲染（`{effectiveOpen && ...}`）——元素瞬间出现/消失。添加 CSS transition：

```css
/* ToolCard body 折叠动画 */
.tool__body,
.tool__nested {
  overflow: hidden;
  transition: max-height 200ms ease, opacity 150ms ease;
  max-height: 600px; /* 展开时足够大 */
  opacity: 1;
}

/* JavaScript 控制折叠时添加这个 class */
.tool--collapsed .tool__body,
.tool--collapsed .tool__nested {
  max-height: 0;
  opacity: 0;
  padding-top: 0;
  padding-bottom: 0;
  margin-top: 0;
  margin-bottom: 0;
}
```

对应修改 `ToolCard.tsx`：不使用条件渲染，改为添加 CSS class 控制可见性，使 transition 生效。

```tsx
// ToolCard.tsx 修改：不用条件渲染，用 CSS class
<div className={`tool__body-area ${effectiveOpen ? "" : "tool--collapsed"}`}>
  {summary && <div className="tool__summary">{summary}</div>}
  {/* ... 其他内容 ... */}
</div>
```

**Reasoning 折叠同理：**
```css
.reasoning__body {
  overflow: hidden;
  transition: max-height 200ms ease, opacity 150ms ease;
  max-height: 2000px;
  opacity: 1;
}
.reasoning__body--collapsed {
  max-height: 0;
  opacity: 0;
}
```

### 2.3 虚拟滚动 + 折叠交互修正
**文件：** `Transcript.tsx`

折叠动画完成后，虚拟滚动需要重新测量高度。使用 `transitionend` 事件：

```tsx
// 在 ToolCard 折叠切换后触发
const onCollapseToggle = useCallback(() => {
  // 等待 CSS transition 完成后重新测量
  setTimeout(() => virtualizer.measure(), 250);
}, [virtualizer]);
```

将这个回调传给 ToolCard，在 `setOpen` 后调用。

### 2.4 验证
- 文本粘贴长 URL/base64 不溢出
- 折叠/展开 ToolCard 有平滑过渡
- 折叠/展开 reasoning 有平滑过渡
- 虚拟滚动在折叠动画后正确更新高度

## 阶段 3：视觉打磨

### 3.1 消息气泡视觉优化
**文件：** `transcript.css`

```css
/* 用户消息：右侧对齐，暖色底色 */
.msg--user .msg__bubble {
  background: var(--accent-soft);
  border: 1px solid var(--accent);
  border-radius: var(--radius) var(--radius) 4px var(--radius);
  align-self: flex-end;
}

/* 助手消息：左对齐，中性底色 */
.msg--assistant .msg__bubble {
  background: var(--bg-soft);
  border: 1px solid var(--border-soft);
  border-radius: var(--radius) var(--radius) var(--radius) 4px;
}
```

### 3.2 工具卡片视觉层级
**文件：** `tool.css`

- 写作工具（`edit_file`、`write_file`）：左侧 accent 色边框
- 只读工具 quiet 模式：更紧凑的间距 + 更低对比度的文字
- 错误状态：更明显的红色左边框

### 3.3 设计 Token 完善
**文件：** `tokens.css`

从 `styles.css` 中提取所有变量并统一命名：
- `--space-*`（间距体系：4/8/12/16/24/32）
- `--radius-*`（圆角体系：4/6/9/12）
- `--shadow-*`（阴影体系）
- `--font-size-*`（字号体系：11/12/13/14/16/18/24）
- 删除重复的两套变量

### 3.4 验证
- `pnpm build` 零错误
- 视觉回归检查（暗色 + 亮色主题）
- 所有面板一致性

## 风险

| 风险 | 级别 | 缓解 |
|------|------|------|
| CSS 拆分导致样式丢失/错位 | 中 | 保持 BEM 类名不变，仅拆分文件；每阶段后构建验证 |
| CSS 变量合并导致颜色错误 | 中 | `search_content` 确认所有变量引用，先搜索再删除 |
| 折叠动画影响虚拟滚动 | 中 | 用 `transitionend` + `setTimeout` 双重保障 |
| tokens.css 两套变量冲突 | 低 | 阶段 1.2 先合并再拆分 |
