## 四、TCCA 四域缓存架构

这是 Tianxuan 的核心架构创新 — 将 LLM 上下文拆分为四个独立缓存域，最大化 DeepSeek 前缀缓存命中率。

### 4.1 架构总览

```
┌──────────────────────────────────────────────────────┐
│                ContextManager (context/manager.go)     │
│                   TCCA V3.0 内核                        │
├──────────────────────────────────────────────────────┤
│ L1: IdentityLayer (cache/identity.go)                 │
│     系统提示词 + 记忆 + 技能索引 + 语言策略               │
│     构建后不可变 → SHA-256 哈希守卫                      │
│     ~3000+ tok · 100% 缓存命中                          │
├──────────────────────────────────────────────────────┤
│ L2: RuntimeLayer (cache/runtime.go)                   │
│     ProjectState + SessionState + ExecutionState      │
│     项目语言/模块/入口 + 编辑记录 + 目标上下文            │
│     首轮锁定 → 会话内稳定                                │
│     ~100-300 tok · 高缓存命中                           │
├──────────────────────────────────────────────────────┤
│ L3: SkillLayer (cache/skill.go)                       │
│     意图分类 + 自适应版本提升                            │
│     根据任务类型选择工具子集 + 策略配置                   │
│     ~1200 tok · 与 L1 耦合                              │
├──────────────────────────────────────────────────────┤
│ L4: FlowLayer (context/flow.go)                       │
│     对话历史 · 每轮追加                                 │
│     自动截断 (≥500K tok → 三层压缩)                      │
│     可变大小 · 新增部分不可缓存                          │
└──────────────────────────────────────────────────────┘
```

### 4.2 L1: IdentityLayer

文件: `internal/cache/identity.go:20`

```go
type IdentityLayer struct {
    systemPrompt string          // 完整 L1 提示词 (不可变)
    context      string          // 预留扩展字段
    registry     *tool.Registry   // 工具注册表 (用于 FilteredSchemas)
}
```

- 构建后**不可变** — 无 Set 方法
- `SaveHash(dir)` / `LoadAndCompareHash(dir)` — 跨会话 SHA-256 一致性验证
- `FilteredSchemas(names)` — 按任务类型提取工具子集 (L3 使用)

**前缀缓存铁律：**
- L1 任何字节变化 → SHA-256 漂移 → **panic** (由 verifyPrefix 守卫)
- tools 列表整个会话不可变 (V8.0.2 教训: filteredSchemas per-turn 导致命中率 0%)
- 系统提示词不能在 session 中途动态修改

### 4.4 L3: SkillLayer

文件: `internal/cache/skill.go:34`


- `Route(input)` — 分类任务意图 → 选择 profile 版本

- `ClassifyFailure(errMsg)` — 区分环境错误 vs 逻辑错误，防止误升级

### 4.5 L4: FlowLayer

文件: `internal/context/flow.go`

- 对话历史管理器，每轮追加 assistant/tool 消息
- 紧凑策略: 上下文窗口 ≥500K tokens 时自动三层压缩
- 子代理独立的 FlowLayer (不继承父代理历史)

### 4.6 ContextManager

文件: `internal/context/manager.go:11`

```go
type ContextManager struct {
    identity *IdentityLayer
    runtime  *RuntimeLayer
    skill    *SkillLayer
    flow     *FlowLayer
    metrics  *CacheMetrics
}
```

核心方法:
- `ProcessFirstTurn(input)` — 首轮: 分类意图 → 锁定 L2 → 返回 TurnContext
- `ProcessTurn()` — 后续轮: 返回当前 TurnContext
- `AssemblePrompt()` — 构建完整 Provider 消息: [L1 system][L2 system][L4 history]
- `Fork(mode, taskPrompt)` — 子代理 fork (继承 L1 + L2 project)

## 七、防御层

### 7.1 三闸门停止系统

文件: `internal/agent/stop_gate.go`

模型输出无工具调用时，必须通过三道闸门才能停止：

| 闸门 | 文件位置 | 检测内容 | 失败处理 |
|------|---------|---------|---------|
| **门1: taskGate** | stop_gate.go:37 | 待办列表是否有未完成项 | 注入针对性 nudge，列出具体未完成项 |
| **门2: goalGate** | stop_gate.go | /goal 目标是否已满足 | 调用独立 judge 模型验证 |
| **门3: verifyGate** | stop_gate.go | orchestrate 模式的最终验证 | 注入验证提示 |

每门有独立重入上限 (cap=3)，超过后 fail-open 允许停止。

| 断路器 | 文件 | 触发条件 | 动作 |

| Batch Storm | agent.go:249 | 整批调用全失败 3 次 | 注入 corrective context |
| Steer | agent.go:959 | 工具调用全失败且 failed≥2 | 注入 mid-turn steer 提示 |
