# Tianxuan V3.0 — TCCA Context Runtime 设计文档

> 状态：RFC · 日期：2026-06-03 · 版本：v3.0.0-draft

## 定位

Tianxuan V3.0 不是一个 Agent 框架，也不是一个 Multi-Agent 框架。

**Tianxuan V3.0 是一个 Cache-First Context Runtime。**

核心创新：把 Context、Cache、Fork、Observability 四者组合成一个完整的运行时内核。Agent 只是运行在这个 Context OS 上的执行进程。

```
ContextManager (内核)
    ↓
AgentRunner (消费者)
    ↓
Controller (适配器)
```

## 设计原则

```
缓存命中率 > 代码美观度
Fork 继承 > 独立上下文
前缀稳定 > 动态灵活
每层独立 > 全局编排
```

## TCCA 四层上下文缓存架构

```
┌─────────────────────────────────────────────────────────────┐
│ L1: Identity Layer                                          │
│ 系统提示 + Agent 规则 + 语言策略 + 项目记忆 + 技能索引        │
│ 规则：构造后不可变。Immutable by construction。               │
│ 缓存：100% 前缀命中。                                        │
├─────────────────────────────────────────────────────────────┤
│ L2: Runtime Layer                                           │
│ ProjectState (锁定后绝对不变)                                 │
│ SessionState (Re​centEdits + ExecutionState，按需序列化)      │
│ Hints (FailureDetector 纠正提示)                             │
│ 规则：ProjectState 首轮 Lock() 后不变；SessionState 可变。     │
│ 缓存：ProjectState 100% 命中；SessionState 变化极小。         │
├─────────────────────────────────────────────────────────────┤
│ L3: Skill Layer                                             │
│ SkillProfile + RouteVersion + Learner 反馈                   │
│ 规则：首轮分类 → 选择 Profile v1。连续失败 → PromoteVersion。  │
│ 缓存：同版本 stable，升级时确定性地变。                        │
├─────────────────────────────────────────────────────────────┤
│ L4: Flow Layer                                              │
│ 对话历史 + DetailRingBuffer + CompactPolicy                  │
│ 规则：仅追加。压缩时替换历史段，保留 ExecutionState。          │
│ 缓存：追加式，仅新增消息 miss。                               │
└─────────────────────────────────────────────────────────────┘
```

### L1: IdentityLayer

```go
type IdentityLayer struct {
    basePrompt     string  // 基础系统提示
    agentRules     string  // TIANXUAN.md
    languagePolicy string  // 语言策略
    projectMemory  string  // MEMORY.md 编译结果
    skillIndex     string  // 技能索引
}

func NewIdentityLayer(basePrompt string, mem *memory.Set, skills []skill.Skill) *IdentityLayer
func (l *IdentityLayer) Bytes() []byte
func (l *IdentityLayer) Fork() *IdentityLayer  // 浅拷贝（共享底层字符串）
```

**不可变规则：** 无 Set 方法。未来即使有人要求 `identity.AddRule()` / `identity.Inject()` / `identity.Append()`，全部拒绝。L1 一旦可变，前缀缓存直接失效。

### L2: RuntimeLayer

```go
type RuntimeLayer struct {
    mu      sync.Mutex
    project ProjectState   // Lock() 后绝对不变
    session SessionState   // 可随时更新
    hints   []string       // 纠正提示
    locked  bool
}

type ProjectState struct {
    Language     string
    Module       string
    EntryPoints  []string
    TopDirs      []string
    TotalFiles   int
    Dependencies []string
}

type SessionState struct {
    RecentEdits   []string
    ActiveModule  string
    CurrentBranch string
    ShellInfo     string
    Execution     ExecutionState  // 短期工作记忆
}

type ExecutionState struct {
    Goal         string
    Phase        string
    ActiveFiles  []string
    Hypothesis   string
    LastAction   string
    PendingTodos []TodoSnap
}

type TodoSnap struct {
    Content string
    Status  string
}
```

**关键约束：**
- ProjectState（Language, Module, Dependencies）首轮 Lock() 后绝不变化 → DeepSeek 100% 缓存命中
- SessionState.RecentEdits 每次编辑后更新，但只在 AssemblePrompt 时序列化到 L2 system message 末尾
- ExecutionState 每步都可能变，变化在 L2 末尾 → DeepSeek 缓存从变化点开始 miss，前面仍命中
- 压缩时保留 ExecutionState.Goal + Phase + PendingTodos；ActiveFiles + Hypothesis 放入 DetailRingBuffer

### L3: SkillLayer

```go
type SkillLayer struct {
    mu       sync.Mutex
    learner  *Learner
    current  SkillProfile
    version  int
}

type SkillProfile struct {
    Kind              TaskKind
    Tools             []string
    PromptHint        string
    Temperature       float64
    MaxSteps          int
    CompactThreshold  float64
    RetryLimit        int
    Verification      VerificationPolicy
}

type VerificationPolicy struct {
    RequireTests    bool
    RequireBuild    bool
    AutoReview      bool
    RequireCitation bool
}
```

**六种 SkillProfile：**

| Kind | v1 工具 | v2 工具 | v3 工具 | Temperature |
|------|---------|---------|---------|-------------|
| FixBug | allButShell | allWithShell | allWithSubagent | 0.3 |
| WriteFeature | allButShell | allWithShell | allWithSubagent | 0.5 |
| Review | readOnly | — | — | 0 |
| Explain | readOnly | — | — | 0 |
| Research | readOnly | allWithSubagent | — | 0.7 |
| Default | allButShell | allWithShell | — | 0.5 |

**RouteVersion 机制：**
- 首轮 `Route(input)` 分类 → 选择 Profile v1
- `RecordOutcome(success)` → 累计 Learner 数据
- `FailureRate > 50%` 或连续 3 次失败 → `PromoteVersion()` → v2
- 再次失败 → v3
- 成功后版本不降级（确定性规则，不重跑分类器）

### L4: FlowLayer

```go
type FlowLayer struct {
    messages []provider.Message
    rings    *DetailRingBuffer   // 压缩详细快照（原名 L2 Ring，改名消除歧义）
    compact  *CompactPolicy
}

type CompactPolicy struct {
    Window     int
    Ratio      float64
    TailTokens int
}

type DetailRingBuffer struct {
    rings []string
    dir   string  // .tianxuan/deep/（文件夹也改名）
    max   int     // 5
}
```

**压缩流程：**
1. 抽取 ExecutionState（Goal, Phase, PendingTodos）
2. 生成 L1 摘要（目标、决策、文件、错误、下一步）
3. 生成详细快照（ActiveFiles, Hypothesis, LastAction, 完整错误 → DetailRingBuffer）
4. 替换 messages = [sys, 摘要, 最近 N 条消息]
5. 重置 ExecutionState 为压缩摘要内容
6. `metrics.RecordCompact(savedTokens)`

## ContextManager — 内核

```go
type ContextManager struct {
    identity *IdentityLayer
    runtime  *RuntimeLayer
    skill    *SkillLayer
    flow     *FlowLayer
    metrics  *CacheMetrics
}
```

**ContextManager 的三个设计约束：**

1. **不持有 Provider 引用。** 上下文管理是纯 CPU 操作，不依赖 LLM。依赖方向：Context → Prompt → Provider，永不 Context → Provider → Context。
2. **不依赖 AgentRunner。** AgentRunner 是消费者。ContextManager 不知道 AgentRunner 的存在。
3. **不依赖 Controller。** Controller 是适配器。ContextManager 不提供会话管理、审批、slash 命令。

**生命周期方法：**

```go
func NewContextManager(identity *IdentityLayer, profile ProjectState, learnerPath string) *ContextManager

// 首轮：分类 → 注入项目状态 → 锁定 L2 → 返回 TurnContext
func (cm *ContextManager) ProcessFirstTurn(input string) *TurnContext

// 后续轮次：L2 已锁定，直接返回上下文
func (cm *ContextManager) ProcessTurn() *TurnContext

// 组装最终 prompt：[L1 system] [L2 project + session + hints] [L3 skill hint] [L4 messages...]
func (cm *ContextManager) AssemblePrompt() []provider.Message

// Fork 子代理上下文
func (cm *ContextManager) Fork(mode ForkMode, taskPrompt string) *ContextManager

// 每轮结束后反馈
func (cm *ContextManager) RecordOutcome(success bool, pricePerToken float64)

// 编辑回调（EditObserver 接口）
func (cm *ContextManager) OnFileEdited(path string)
func (cm *ContextManager) OnExecutionUpdated(es ExecutionState)

// 查询
func (cm *ContextManager) ActiveTools() []provider.ToolSchema
func (cm *ContextManager) CurrentProfile() SkillProfile
func (cm *ContextManager) Metrics() CacheReport
```

## Fork 继承

```go
type ForkMode int
const (
    ForkIndependent   ForkMode = iota  // Type A: 纯独立任务
    ForkCollaborative                   // Type B: 继承工作记忆（Phase 6+）
)

func (cm *ContextManager) Fork(mode ForkMode, taskPrompt string) *ContextManager {
    child := &ContextManager{
        identity: cm.identity.Fork(),     // L1: 浅拷贝共享
        metrics:  cm.metrics.NewChild(),  // 分层统计
    }

    // L2: 共享 project，session 根据模式决定
    child.runtime = &RuntimeLayer{
        project: cm.runtime.project,
        session: mode == ForkCollaborative ? cm.runtime.session.Clone() : SessionState{},
        hints:   nil,
    }

    // L3: 独立分类
    child.skill = NewSkillLayer(cm.skill.learner)

    // L4: 完全独立
    child.flow = NewFlowLayer(cm.flow.compact)

    return child
}
```

**继承矩阵：**

| 层 | Independent | Collaborative |
|----|------------|---------------|
| L1 Identity | ✅ 共享 | ✅ 共享 |
| L2 ProjectState | ✅ 共享 | ✅ 共享 |
| L2 SessionState | ❌ 独立 | ✅ 继承 |
| L2 Hints | ❌ 独立 | ❌ 独立 |
| L3 Skill | ❌ 独立 | ❌ 独立 |
| L4 Flow | ❌ 独立 | ❌ 独立 |

## AgentRunner — 消费者（14 个顶层字段）

```go
type AgentRunner struct {
    prov       provider.Provider
    ctxMgr     *ContextManager       // 唯一上下文入口
    tools      *tool.Registry

    sink       event.Sink
    mode       RunnerMode
    planner    *Planner

    dispatcher *ToolDispatcher
    gate       Gate
    hooks      ToolHooks

    storm      *StormBreaker
}
```

**相比 V2.4 (38 字段) 移除的字段：**
- 8 个压缩字段 → `ctxMgr.flow.compact`
- 2 个 L2 环字段 → `ctxMgr.flow.rings`
- 2 个 schema 过滤字段 → `ctxMgr.ActiveTools()`
- 2 个 learner 字段 → `ctxMgr.skill.RecordOutcome()`
- `runtimePrompt` → `ctxMgr.AssemblePrompt()`
- `session` + `sessMu` → 仍保留（Agent 需要直接操作消息）

## Controller — 纯会话编排

```go
type Controller struct {
    ctxMgr    *ContextManager
    runner    agent.Runner

    sessionDir string
    firstTurn  bool
    autoPlan   string
    classifier autoPlanClassifier

    sink     event.Sink
    host     *plugin.Host
    commands []command.Command
    skills   []skill.Skill
    hooks    *hook.Runner
    jobs     *jobs.Manager
}
```

**runTurn 首轮处理简化为一行：**

```go
func (c *Controller) runTurn(ctx context.Context, input string) error {
    c.maybeSessionStart(ctx)
    c.maybeAutoPlan(ctx, input)
    if !c.firstTurn {
        c.ctxMgr.ProcessFirstTurn(input)
        c.firstTurn = true
    }
    return c.runner.Run(ctx, input)
}
```

当前 V2.4 中 `runTurnWithRaw` 的 20 行首轮分类代码（Route → FilterSchemas → SetActiveSchemas → ApplyRoute → Lock → SetRuntimePrompt → SetTaskKind）——全部消失。

## 完整调用链

```
boot.Build(cfg)
  ├─ profile.Scan(cwd)  →  ProjectState
  ├─ identity = NewIdentityLayer(basePrompt, mem, skills)
  ├─ ctxMgr = NewContextManager(identity, profile, learnerPath)
  │    ├─ NewRuntimeLayer(projectState)
  │    ├─ NewSkillLayer(learner)
  │    └─ NewFlowLayer(compactPolicy)
  ├─ dispatcher = NewToolDispatcher(nil, gate, hooks)
  │    └─ dispatcher.observer = ctxMgr  // EditObserver 注入
  ├─ runner = NewAgentRunner(prov, ctxMgr, tools, dispatcher, ...)
  └─ ctrl = NewController(ctxMgr, runner, ...)

User Input → Controller.runTurn(input)
  ├─ ctxMgr.ProcessFirstTurn(input)        // 仅首轮
  │    ├─ skill.Route(input)                // L3: FixBug-v1
  │    ├─ runtime.Lock()                    // L2: 项目状态锁定
  │    └─ → TurnContext(SystemPrompt, Tools, Profile)
  └─ runner.Run(input)
       ├─ ctxMgr.AssemblePrompt()           // L1 + L2 + L3 + L4
       ├─ stream(provider, prompt, tools)
       ├─ executeOne()
       │    ├─ dispatcher.Check()            // planMode → gate → hooks
       │    ├─ tool.Execute()
       │    └─ dispatcher.NotifyEdit(path)   // → ctxMgr.OnFileEdited()
       │         └─ runtime.TrackEdit()       // → SessionState.RecentEdits
       ├─ flow.Add(result)
       └─ flow.MaybeCompact(usage, execState)

  ctxMgr.RecordOutcome(success, pricePerToken)
    ├─ skill.RecordOutcome() → learner
    │    └─ failureRate > 50% → skill.PromoteVersion()
    └─ metrics.RecordCompact(savedTokens, price)
```

## 五阶段迁移路线

| 阶段 | 内容 | 风险 | 文件影响 |
|------|------|------|----------|
| **Phase 1** | GoalRouter 精度修复 + Controller 重复代码消除 | 低 | cache/router.go, control/controller.go |
| **Phase 2** | Compiler→IdentityLayer + RuntimeContext→RuntimeLayer | 中 | cache/ 包重构 |
| **Phase 3** | GoalRouter+Learner→SkillLayer+SkillProfile+RouteVersion | 高 | cache/ 核心行为变更 |
| **Phase 4** | AgentRunner 字段分组 + DetailRingBuffer 重命名 | 低 | agent/ 结构整理 |
| **Phase 5** | ContextManager 统一 + AgentRunner 移除缓存状态 + EditObserver | 高 | 跨包重构，cache/→context/ |
| **Phase 6+** | CacheMetrics 货币化 + ForkCollaborative + `/tcca stats` | 中 | 新功能 |
