# V1.5 缓存命中率测算 — 测试计划

> **目标**：在真实 DeepSeek API 上对比 V1.4 vs V1.5 的缓存命中率，验证三层重构效果。
> **基准**：V1.4 命中率 ~94%（miss 6%），目标 V1.5 ≥99.3%（miss ≤0.7%）。

---

## 1. 测算原理

DeepSeek API 返回的 `usage` 对象包含两个关键字段：

```json
{
  "prompt_cache_hit_tokens":  12345,   // 从缓存读取的 token 数
  "prompt_cache_miss_tokens": 678      // 未命中缓存、按原价计费的 token 数
}
```

**命中率** = `hit_tokens / (hit_tokens + miss_tokens) × 100%`

每次 API 调用的 usage 被 `agent.go:stream()` 记录到 `sessCacheHit` / `sessCacheMiss`，并通过 `event.Usage` 事件上报。`StatusBar` 组件实时展示单轮和聚合命中率。

## 2. 测试场景

### 场景 A：单会话多轮对话（测量会话内缓存稳定性）

模拟用户的典型使用：同一个会话中连续发送 N 个问题。

| 参数 | 值 |
|------|-----|
| 轮数 | 15 轮 |
| 每轮输入 | 不同主题的代码问题（`fix bug` / `explain` / `add feature` 交替）|
| 预期 | 首轮 miss → 后续轮 prefix 稳定 → 命中率逐步攀升至 >95% |

**测量指标**：
- 每轮 API 的 `prompt_cache_hit_tokens` / `prompt_cache_miss_tokens`
- 会话聚合命中率（`SessionCache()`）
- 与 V1.4 同场景对比

### 场景 B：工具循环（测量工具调用期间的缓存）

模拟密集工具调用场景，测量 assistant(tool_calls) + tool(result) 追加后的前缀稳定性。

| 参数 | 值 |
|------|-----|
| 输入 | `"读取 internal/cache/compiler.go，分析其缓存架构，如果发现问题请直接修复"` |
| 预期轮数 | 5-10 轮（含多次 read_file → edit_file）|
| 预期 | 工具循环中每轮 API 调用的系统前缀命中率 ≥95% |

**测量指标**：
- 每轮 API 的缓存命中率
- 是否有因 compaction 导致的命中率骤降

### 场景 C：子 Agent 派生（测量 Fork 后的缓存继承）

测量 `task` 工具派生子 Agent 时的缓存命中。

| 参数 | 值 |
|------|-----|
| 输入 | `"使用 task 工具派生子 agent 审查 internal/cache/ 目录的线程安全性"` |
| 预期 | 子 Agent 首轮 API 调用命中父 Agent 的 L1 前缀 |

**测量指标**：
- 子 Agent 首轮 prompt 的缓存命中率（应 >80%，因为 L1 前缀共享）

## 3. V1.4 vs V1.5 对比方案

### 3.1 执行方式

```bash
# V1.4 基准测试
git checkout v1.4.0
go build -o bin/tianxuan-v1.4.exe ./cmd/tianxuan
# 运行场景 A/B/C，收集 usage 数据

# V1.5 对比测试
git checkout v1.5-dev
go build -o bin/tianxuan-v1.5.exe ./cmd/tianxuan
# 运行相同场景，收集 usage 数据
```

### 3.2 自动化脚本设计

编写 `scripts/cache-bench.ps1`（PowerShell 脚本），自动执行以下流程：

```
1. 启动 tianxuan headless 模式
2. 逐轮发送预设输入
3. 从 JSONL session 文件中提取每轮 API 的 usage
4. 输出 CSV 格式的对比报告
```

### 3.3 CSV 报告格式

```csv
version,scenario,turn,prompt_tokens,hit_tokens,miss_tokens,hit_rate,aggregate_rate
v1.4,dialogue,1,1200,0,1200,0%,0%
v1.4,dialogue,2,3500,1200,2300,34%,17%
...
v1.5,dialogue,1,1100,0,1100,0%,0%
v1.5,dialogue,2,3400,1100,2300,32%,16%
```
